# Maximum Subarray Sum Not Exceeding B

## Problem Statement

You are given:

- An integer `A` representing the size of the array.
- An integer `B` representing the maximum allowed sum.
- An integer array `C` of size `A`.

Your task is to find the **maximum possible sum of a contiguous subarray** such that the sum **does not exceed** `B`.

Return the maximum valid subarray sum.

---

## Example

### Input

```text
A = 5

B = 12

C = [2, 1, 3, 4, 5]
```

### Output

```text
12
```

### Explanation

Possible subarrays include:

| Subarray | Sum |
|----------|----:|
| [2] | 2 |
| [2, 1] | 3 |
| [2, 1, 3] | 6 |
| [2, 1, 3, 4] | 10 |
| [2, 1, 3, 4, 5] | 15 ❌ |
| [3, 4, 5] | 12 ✅ |
| [4, 5] | 9 |

The maximum subarray sum that does not exceed `12` is:

```text
12
```

---

## Another Example

### Input

```text
A = 4

B = 8

C = [5, 2, 1, 6]
```

### Output

```text
8
```

### Explanation

The subarray:

```text
[5, 2, 1]
```

has a sum of:

```text
8
```

which is the maximum possible sum that does not exceed `B`.

---

## Approach

The solution uses a **brute-force approach** to examine every possible subarray.

For every starting index:

1. Extend the subarray one element at a time.
2. Maintain a running sum.
3. Whenever the running sum is less than or equal to `B`, update the maximum answer.

Using a running sum avoids recalculating the sum of the same subarray repeatedly.

---

## Algorithm

1. Initialize `answer = 0`.
2. Traverse every possible starting index.
3. Initialize `sum = 0`.
4. Extend the ending index one step at a time.
5. Add the current element to the running sum.
6. If:

```text
sum ≤ B
```

update the answer.

7. Return the maximum valid sum.

## Why Does This Work?

Every contiguous subarray has:

- A starting index.
- An ending index.

The outer loop selects the starting index.

The inner loop expands the ending index while maintaining the running sum.

Whenever the current sum is less than or equal to `B`, it becomes a valid candidate for the answer.

Since every possible subarray is examined exactly once, the algorithm is guaranteed to find the maximum valid sum.

---

## Complexity Analysis

### Time Complexity

There are approximately:

```text
N × (N + 1) / 2
```

possible subarrays.

Each subarray is processed using a running sum.

Overall:

```text
O(N²)
```

---

### Space Complexity

Only a few variables are used.

```text
O(1)
```

---

## Java Solution

```java
public class Solution {

    public int maxSubarray(int A, int B, int[] C) {

        int n = C.length;

        int ans = 0;

        for (int s = 0; s < n; s++) {

            int sum = 0;

            for (int e = s; e < n; e++) {

                sum += C[e];

                if (sum <= B) {
                    ans = Math.max(ans, sum);
                }
            }
        }

        return ans;
    }
}
```

---

## Key Learning

- Every subarray can be generated by choosing a starting index and expanding the ending index.
- Maintaining a running sum avoids repeatedly calculating the sum of the same subarray.
- This reduces the brute-force solution from **O(N³)** to **O(N²)**.
- The algorithm checks every possible contiguous subarray and updates the answer only when the sum is less than or equal to `B`.
- This problem demonstrates how incremental computation (running sums) can significantly improve the efficiency of brute-force subarray algorithms.
