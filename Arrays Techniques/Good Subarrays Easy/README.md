# Count Good Subarrays

## Problem Statement

You are given:

- An integer array `A` of size `N`.
- An integer `B`.

A subarray is considered **good** if it satisfies **either** of the following conditions:

1. The length of the subarray is **even**, and the sum of its elements is **less than `B`**.
2. The length of the subarray is **odd**, and the sum of its elements is **greater than `B`**.

Your task is to count the total number of **good subarrays**.

Return the count.

## Approach

The solution uses a **brute-force approach** to examine every possible subarray.

For every starting index:

1. Extend the ending index one element at a time.
2. Maintain a running sum.
3. Calculate the current subarray length.
4. Check whether the subarray satisfies either of the required conditions.
5. If it does, increment the count.

Maintaining a running sum avoids recalculating the sum for every subarray.

---

## Algorithm

1. Initialize the answer as `0`.
2. Traverse every possible starting index.
3. Initialize the running sum as `0`.
4. Expand the ending index.
5. Update:
   - Current subarray length.
   - Running sum.
6. If:
   - Length is even **and** sum is less than `B`, increment the count.
   - Length is odd **and** sum is greater than `B`, increment the count.
7. Continue until all subarrays have been checked.
8. Return the total count.

## Why Does This Work?

Every contiguous subarray can be uniquely identified by its starting and ending indices.

The algorithm:

- Generates every possible subarray.
- Maintains a running sum while extending the subarray.
- Determines the current length using:

```text
Length = End Index - Start Index + 1
```

It then checks the problem conditions:

- **Even-length** subarrays must have a sum **less than `B`**.
- **Odd-length** subarrays must have a sum **greater than `B`**.

Since every possible subarray is examined exactly once, the algorithm correctly counts all good subarrays.

---

## Complexity Analysis

### Time Complexity

The total number of subarrays is:

```text
N × (N + 1) / 2
```

Each subarray is processed once using a running sum.

Overall:

```text
O(N²)
```

---

### Space Complexity

Only a few variables are used.

```text
O(1)
```

## Java Solution

```java
public class Solution {

    public int solve(int[] A, int B) {

        int n = A.length;

        int count = 0;

        for (int s = 0; s < n; s++) {

            int sum = 0;

            for (int e = s; e < n; e++) {

                int length = e - s + 1;

                sum += A[e];

                if (length % 2 == 0 && sum < B) {
                    count++;
                } else if (length % 2 != 0 && sum > B) {
                    count++;
                }
            }
        }

        return count;
    }
}
```

---

## Key Learning

- Every contiguous subarray can be generated by selecting a starting index and expanding the ending index.
- Maintaining a running sum avoids recomputing the sum for each subarray, improving the brute-force approach from **O(N³)** to **O(N²)**.
- The parity (even or odd) of a subarray can be determined directly from its length using `(end - start + 1)`.
- Problems involving multiple conditions can often be solved by evaluating each generated subarray against the required criteria.
- This solution runs in **O(N²)** time with **O(1)** extra space.
